## 0. BLUF（Bottom Line Up Front）

本 Spec 把 Research v1.0 中已通过 13 项 TBD 闭环的"3T + Backseat Driver + 多时间尺度混合避碰"架构，固化为一个对接 CCS《智能船舶规范》(2025) 智能系统认可路径、可执行、可入级的战术决策层设计。**核心差别**：白盒主决策（MPC + COLREGs 形式化规则）+ 短期 BC-MPC 兜底 + 模式状态机 D2/D3/D4/MRC + 5 组在 v1 即冻结的接口契约 + 船舶动力学插件化。**核心权衡**：在 CCS 优先 + 安全优先（船舶项目优先级 安全 > 合规 > 可维护 > 性能 > UX）下，主动放弃端到端 DRL / 黑盒决策，以换取认证可审与多船型迁移成本可控；FCB 高速段通过 MMG + 高速修正 + Robust MPC 处理水动力不确定性，而非依赖闭环辨识。

**本 Spec 核心架构 / 算法**：

- **架构范式**：Three-Tier（战略 / 战术 / 控制）+ Backseat Driver 解耦 + 模式管理贯通竖列
- **避碰算法**：中程 MPC（3-5 min 预测时域，COLREGs Rule 5/6/7/8/13-19 约束嵌入）+ 短期 BC-MPC（30-60 s 预测时域，多分支兜底）+ 指令仲裁器
- **模式管理**：D2 ↔ D3 ↔ D4 + MRC 兜底 + ROC override，三档接管时间预算（紧急 ≤ 30 s / 监督 ≤ 60 s / 非紧急 ≤ 5 min）作为初始设计基线
- **多船型支持**：船舶动力学插件化（MMG 3-DOF 基底 + 船型特化修正项），FCB 首发用 Yasukawa 高速修正
- **AI 边界**：主决策路径全白盒；ML 仅限辅助子模块（他船意图预测、感知预处理、传感器融合下游），需重型 V&V

**实装范围**：

- 战术层 7 个核心模块 + 1 个模式管理器的接口契约、状态机、降级矩阵
- 5 组对外接口（战略 / 控制 / ROC / 感知融合 / 公共服务）的 v1 字段冻结
- CCS 智能系统认可的提交件清单（每个子模块独立认可路径）
- COLREGs Rule 5/6/7/8/13-19 + Rule 2 的形式化处理边界
- 各模块的 SIL 候选分配（最终在 HAZID 后冻结）
- **V&V 验证基础设施**（v1.1 新增）：SIL 分配方法学（STPA + Software FMEA + BN）、X-in-the-loop 测试金字塔（MIL/SIL/HIL/Sea Trial）、场景生成方法学（AIS 数据驱动 + 形式化 STPA→STL→GP + ML 辅助）、仿真器选型路径（用户技术栈倾向 Unreal/Unity 高保真渲染）、场景描述语言（v1 自定义 YAML schema）

**不在本 Spec 范围内**：

- 各模块内部具体算法实现细节（在 Plan 阶段展开为 task 级）
- 感知融合内部算法（属感知层）
- 控制层内部算法（PID/NMPC track control 属控制层）
- 战略层航次规划具体算法
- 网络安全实现（IACS UR E26/E27 + CCS《船舶网络安全指南》单独 Spec）
- CCS 入级文档正式提交流程（项目管理范畴）

## 1. 概述与职责定义

### 1.1 模块定位

WP Generator（航点生成器）是 MASS_ADAS 五层架构中 Layer 2（Voyage Planner，航路规划层）的核心子模块。它将 Mission Layer 给出的粗粒度航次计划（起终点和少量途经点）转化为精细化的航线——由一系列经纬度航点构成，每个航点附带完整的操纵参数元数据。

### 1.2 核心职责

WP Generator 的根本职责是：**确保输出的航线上的每一个转弯、每一段直航、每一次经过受限水域，都在本船的物理能力包络之内。** 这等价于对船长"航前审图"工作的完整数字化。

### 1.3 设计原则

- **保守原则：** 所有安全余量取保守值。宁可降速/绕路，不可逼近物理极限。
- **可标定原则：** 所有经验值和阈值均从 Parameter DB 读取，不硬编码。海试数据到位后可直接替换。
- **可回溯原则：** 每个航点的生成理由（为何在此处、为何此速度、为何此转角）应记录在航点元数据中，便于审计和调试。
- **分层原则：** WP Generator 不直接控制船舶。它只输出航点序列和约束参数，由下游的 Guidance 层（L4）负责实时跟踪执行。

## 3. 核心参数数据库

### 3.1 旋回特性参数表

旋回特性随速度变化。Parameter DB 中应存储一组离散速度点下的旋回参数，运行时通过线性插值获取任意速度下的值。

|-------------|------------|-------------|-------------|----------------|-------------------|----------------|
| 2.0 | 3.9 | 待标定 | 待标定 | 待标定 | 待标定 | 待标定 |
| 4.0 | 7.8 | 待标定 | 待标定 | 待标定 | 待标定 | 待标定 |
| 6.0 | 11.7 | 待标定 | 待标定 | 待标定 | 待标定 | 待标定 |
| 8.0 | 15.6 | 待标定 | 待标定 | 待标定 | 待标定 | 待标定 |
| 10.0 | 19.4 | 待标定 | 待标定 | 待标定 | 待标定 | 待标定 |

**海试数据到位前的经验初值（基于 Clarke 1983 回归公式）：**

对于常规排水型船舶，可用以下经验公式做初始估算：

```
A_d ≈ (3.0 + 2.0 × Cb) × L        （Cb 为方形系数）
T_r ≈ (1.5 + 1.0 × Cb) × L
D_T ≈ (4.0 + 2.0 × Cb) × L
D_s ≈ 0.85 × D_T
k_v ≈ 0.6 + 0.2 × (1 - Cb)
```

其中 L 为船长（m），Cb 为方形系数（船体饱满程度，范围 0.4~0.85）。USV 的 Cb 通常较低（0.4~0.55），意味着操纵性较好、旋回半径相对较小。

**重要警告：** 以上经验公式仅适用于低速排水状态。USV 在高速滑行状态下的水动力特性与排水状态完全不同——船体升出水面、湿面积急剧减小、侧向阻力大幅降低——旋回半径可能显著增大。高速旋回参数必须通过实船试验或高精度 CFD 获取，经验公式不可靠。

## 4. 计算流程总览

WP Generator 的完整计算流程分为六个步骤，按顺序执行：

```
输入：粗粒度航次计划（3~5 个途经点）
          │
          ▼
    ┌─────────────────────┐
    │ 步骤一：初始航点序列生成 │ ← ENC 海图数据
    │ 在途经点间插入精细航点   │
    │ 确保航线不穿越禁区/陆地  │
    └──────────┬──────────┘
               │
               ▼
    ┌─────────────────────┐
    │ 步骤二：转角评估        │ ← 旋回特性参数
    │ 每个航点计算 Δψ        │
    │ 评估空间占用 vs 可用走廊 │
    └──────────┬──────────┘
               │
          ┌────┴────┐
          │ 可行？   │
          ├─ 是 ────→ 进入步骤四
          └─ 否 ────→ 步骤三
               │
               ▼
    ┌─────────────────────┐
    │ 步骤三：不可行转弯处理   │
    │ 策略 A：降低计划速度    │
    │ 策略 B：拆分为多个小弯   │
    │ 策略 C：请求 L1 重规划   │
    └──────────┬──────────┘
               │
               ▼
    ┌─────────────────────┐
    │ 步骤四：WOP 计算       │ ← 旋回参数 + 舵系统参数
    │ 计算每个转弯的转舵提前量 │
    └──────────┬──────────┘
               │
               ▼
    ┌─────────────────────┐
    │ 步骤五：航点间距校验     │
    │ 检查最小/最大间距约束    │
    │ 自动插入或合并航点      │
    └──────────┬──────────┘
               │
               ▼
    ┌─────────────────────┐
    │ 步骤六：特殊水域航点加密  │ ← ENC 水域分类
    │ 进出港/海峡/TSS/锚地    │
    │ 加密航点 + 应用限速规则   │
    └──────────┬──────────┘
               │
               ▼
输出：精细化航点序列 + 速度剖面
```

## 5. 步骤一：初始航点序列生成

### 5.1 算法描述

将 Mission Layer 给出的粗粒度途经点（通常 3~5 个）之间用直线段连接，然后对每一段执行以下检查：

1. **陆地穿越检查：** 航线段是否与海岸线多边形相交？
2. **禁航区检查：** 航线段是否进入任何禁航区/限航区多边形？
3. **TSS 穿越检查：** 航线段是否横穿交通分隔带（应沿分道方向航行）？
4. **水深检查：** 航线段沿线的最浅水深是否满足最小 UKC 要求？

### 5.2 空间查询实现

使用 R-tree 空间索引对 ENC 要素进行预索引，查询复杂度降至 O(log n)。推荐使用 Boost.Geometry R-tree（C++）或 rtree 库（Python）。

### 5.3 安全距离标准

| 要素类型 | 最小安全距离 | 说明 |
|---------|------------|------|
| 海岸线（沿岸航行） | ≥ 2.0 nm | 大型船舶增至 3~5 nm |
| 海岸线（USV 近岸作业） | ≥ 0.5 nm | 需充分考虑浅水效应 |
| 孤立危险物 | ≥ 2 × 等深线范围 | 例如：暗礁周围 10m 等深线半径 200m，则安全距离 ≥ 400m |
| 沉船 | ≥ 0.5 nm | 周围可能有渔网/锚链 |
| 禁航区边界 | ≥ 0.5 nm | 留出偏流裕度 |
| TSS 分隔带边界 | ≥ 0.25 nm | 沿分道方向航行时的横向距离 |
| 其他船舶锚泊区边界 | ≥ 0.5 nm | 锚泊船可能走锚 |

### 5.4 航线段不可行时的处理

当检测到航线段与禁区相交时，执行以下步骤：

1. 计算航线段与禁区多边形的交点。
2. 沿禁区多边形的外侧边界（保持安全距离）生成一组绕行航点。
3. 选择绕行距离最短的方向（左绕或右绕）。
4. 将绕行航点插入航线序列中。

### 5.5 最小 UKC（龙骨下富余水深）标准

| 水域类型 | 最小 UKC |
|---------|---------|
| 开阔水域 | ≥ 吃水 T 的 50%，且不小于 3.0 m |
| 沿岸航行 | ≥ 吃水 T 的 30%，且不小于 2.0 m |
| 港内/航道 | ≥ 吃水 T 的 15%，且不小于 1.0 m（需考虑 Squat 效应） |
| 锚地 | ≥ 吃水 T 的 100%，且不小于 5.0 m |

**Squat 效应修正：** 在浅水受限航道中，船速引起的船体下沉量（Squat）必须从 UKC 中扣除。Squat 量的经验公式（Barrass 简化公式）：

```
S_max = Cb × (V_k²) / 100
```

其中 S_max 为最大下沉量（米），Cb 为方形系数，V_k 为船速（节）。

示例：Cb = 0.50 的 USV 以 10 节航行，S_max = 0.50 × 100 / 100 = 0.50 m。即龙骨实际比静止时下沉 0.50 米，可用 UKC 减少 0.50 米。

## 6. 步骤二：转角评估算法

### 6.1 转角计算

对于航点序列 WP_{i-1} → WP_i → WP_{i+1}，WP_i 处的转角 Δψ_i 计算如下：

```
φ_1 = atan2(WP_i.lon - WP_{i-1}.lon, WP_i.lat - WP_{i-1}.lat)    # 前一航段方位角
φ_2 = atan2(WP_{i+1}.lon - WP_i.lon, WP_{i+1}.lat - WP_i.lat)    # 后一航段方位角

Δψ_i = φ_2 - φ_1
```

将 Δψ_i 归一化至 [-180°, +180°] 区间。正值表示右转，负值表示左转。绝对值 |Δψ_i| 即为转角大小。

**注意：** 以上使用简化的平面计算。当航点间距超过 50 nm 时，应使用大圆航线的初始/终止方位角来计算 Δψ，否则会有不可忽略的误差。大圆方位角计算使用 Vincenty 公式或 Haversine 公式。

### 6.2 旋回空间占用计算

给定 WP_i 处的转角 Δψ 和当前计划速度 V，从 Parameter DB 查表（线性插值）获取该速度下的旋回参数：

```
A_d(V)  : 纵距
T_r(V)  : 横距
D_T(V)  : 战术直径
D_s(V)  : 稳定旋回直径
R(V)    : 稳定旋回半径 = D_s(V) / 2
```

旋回过程中船的轨迹是一条复杂的渐开线（从直线过渡到圆弧再回到直线），精确计算需要求解旋回运动方程。但在规划阶段，可以使用以下几何近似模型：

**外侧最大横向偏移量 Y_max（航线外侧的空间占用）：**

```
当 |Δψ| ≤ 90° 时:
    Y_max(Δψ, V) = R(V) × (1 - cos(Δψ)) + T_r(V) × sin(Δψ) × (|Δψ| / 90°)
    
    简化近似（精度约 ±10%）：
    Y_max ≈ T_r(V) × (|Δψ| / 90°)

当 90° < |Δψ| ≤ 180° 时:
    Y_max(Δψ, V) = R(V) × (1 - cos(Δψ)) + 纵距修正项
    
    简化近似：
    Y_max ≈ T_r(V) + (A_d(V) - T_r(V)) × ((|Δψ| - 90°) / 90°)
```

**纵向空间占用 X_max（沿原航向方向的超前距离）：**

```
当 |Δψ| ≤ 90° 时:
    X_max ≈ A_d(V) × (|Δψ| / 90°)

当 90° < |Δψ| ≤ 180° 时:
    X_max ≈ A_d(V) + R(V) × sin(|Δψ| - 90°)
```

### 6.3 精确旋回轨迹模型（可选高精度实现）

如果简化公式的精度不能满足需求（特别是在极其受限的水域），可以使用参数化旋回轨迹模型。旋回轨迹的参数方程为：

```
进入段（打舵后到建立稳定旋回前，约 1~2 个船长的过渡段）:
    x(t) = V × t - (V × t³) / (6 × R² × T_r_time²)
    y(t) = (V × t²) / (2 × R × T_r_time)
    其中 T_r_time 为建立旋回的时间常数（从 Z 形试验获得）

稳定旋回段:
    x(t) = x_0 + R × sin(ω × t + φ_0)
    y(t) = y_0 + R × (1 - cos(ω × t + φ_0))
    其中 ω = V_turn / R, V_turn = V × k_v（考虑速降后的旋回速度）

恢复段（回舵后到恢复直航，与进入段对称）
```

在实际工程中，建议先用简化公式实现 V1 版本，用仿真验证其精度是否足够，不足时再升级为精确模型。

### 6.4 可行性判定

定义可用安全走廊半宽 W_avail(WP_i)：以 WP_i 为中心，向航线两侧扩展，直到遇到以下约束之一为止的距离：

- 水深等值线（水深 < 本船吃水 + 最小 UKC）
- 障碍物安全距离圈
- 航道边界 / TSS 边界
- 其他航线的安全间距

**安全系数：** 实际使用的走廊宽度应乘以安全系数 η：

```
W_usable = W_avail × η

η 取值：
  开阔水域，气象良好：η = 0.70（留 30% 余量）
  沿岸/受限水域：η = 0.60（留 40% 余量）
  港内/极受限水域：η = 0.50（留 50% 余量）
  恶劣天气（风力 > BF6 或能见度 < 2nm）：η 再减小 0.10
```

**判定条件：**

```
IF Y_max(Δψ, V) ≤ W_usable(WP_i) THEN
    → 可行。记录并进入步骤四。
ELSE
    → 不可行。进入步骤三。
```

## 7. 步骤三：不可行转弯处理策略

当步骤二判定转弯不可行时，按以下优先级依次尝试三种策略。

### 7.1 策略 A：降低计划速度

**目标：** 找到使该转弯刚好可行的最大安全速度 V_safe。

**算法：** 二分搜索法。

```
V_low  = V_min_maneuvering    # 最低可操纵速度（低于此速度舵效不足）
V_high = V_current            # 当前计划速度
tolerance = 0.1               # m/s

WHILE (V_high - V_low) > tolerance:
    V_mid = (V_low + V_high) / 2
    Y_test = Y_max(Δψ, V_mid)  # 用 V_mid 重新计算空间占用
    IF Y_test ≤ W_usable:
        V_low = V_mid          # V_mid 可行，尝试更高速度
    ELSE:
        V_high = V_mid         # V_mid 不可行，降低速度

V_safe = V_low                # 最终安全速度（保守取 V_low）
```

**约束条件：**

- V_safe 不得低于最低可操纵速度 V_min_maneuver。对于依赖舵效的常规船舶，这个速度通常为 3~4 节（1.5~2.0 m/s）。对于配备侧推器或差动推进的 USV，这个下限可以更低。
- V_safe 需传递给 Speed Profiler，由其在上游航段生成减速曲线（详见第 11 节）。

**如果 V_safe < V_min_maneuver，策略 A 失败，进入策略 B。**

### 7.2 策略 B：转弯拆分

**目标：** 将一个大转角拆分为若干个小转角，每个小转角各自满足空间约束。

**最大单次转角限制（经验值）：**

| 水域类型 | 建议最大单次转角 |
|---------|---------------|
| 开阔水域 | ≤ 60° |
| 沿岸/受限水域 | ≤ 45° |
| 港内 | ≤ 30° |
| 极受限水域（窄航道转弯） | ≤ 20° |

**拆分算法：**

输入：原航点 WP_B（位于 WP_A 和 WP_C 之间），转角 Δψ，目标子转角 Δψ_sub。

```
n = ceil(|Δψ| / Δψ_sub)     # 需要拆分成 n 段
Δψ_actual = |Δψ| / n         # 每段实际转角

# 计算拆分偏移距离 d
# d 为新航点偏离原航线交叉点的距离
# 使得 n 个子转角依次完成后总转角等于 Δψ

d = R(V_safe) / tan(Δψ_actual / 2)

# 生成 n-1 个过渡航点
# 沿原航段 AB 的延长线和 BC 的起始方向之间均匀分布

FOR k = 1 TO n-1:
    θ_k = φ_1 + k × Δψ_actual × sign(Δψ)    # 第 k 个过渡点的方位角
    WP_B_k.lat = WP_B.lat + d × cos(θ_k) × scale_factor
    WP_B_k.lon = WP_B.lon + d × sin(θ_k) × scale_factor
    
    # 验证新航点的安全性（水深、障碍物）
    IF NOT safe(WP_B_k):
        → 策略 B 失败，进入策略 C
```

**拆分后验证：** 每个子转角必须重新经过步骤二的转角评估。如果任何一个子转角仍然不可行，需要进一步降速或增加拆分数量。

### 7.3 策略 C：请求重规划

当策略 A 和策略 B 均失败时，说明在当前的航线走向下，该转弯在物理上不可执行（即使以最低速度和最小子转角也无法满足空间约束）。

此时 WP Generator 应：

1. 记录失败原因和约束参数。
2. 向 Mission Layer（L1）发送重规划请求（通过 ROS2 Service `/decision/voyage/replan`）。
3. 请求中附带建议："该转弯处需要至少 XX 米的安全走廊，或需要调整途经点以避免此转角"。

## 8. 步骤四：转舵点（WOP）计算

### 8.1 WOP 基本公式

对于通过评估的转弯（转角 Δψ，执行速度 V_turn），WOP 的位置定义为：**船在到达航点之前多远的距离处开始打舵，使得旋回弧线的终点正好切入新航段方向。**

```
WOP_base = A_d(V_turn) × (|Δψ| / 90°) - R(V_turn) × sin(|Δψ|)

当 |Δψ| ≤ 90° 时，简化为:
    WOP_base ≈ R(V_turn) × tan(|Δψ| / 2)
```

### 8.2 WOP 修正量

基本公式假设舵角瞬间到位且船体立即响应。实际中需要加上以下修正量：

**舵响应延迟修正：**

```
WOP_rudder = V_turn × (τ_δ + t_δ / 2)

其中：
  τ_δ  = 舵响应延迟（从下令到舵开始动作），典型值 1~3 秒
  t_δ  = 舵角建立时间（从 0° 到目标舵角），典型值 5~14 秒（取决于目标舵角）
  V_turn = 转弯时的实际船速
```

**船体惯性修正（经验项）：**

```
WOP_inertia = k_wop × L

其中：
  k_wop = 1.0~2.0（经验系数，操纵性好的船取小值，操纵性差的船取大值）
  L     = 船长
```

**风流修正：** 如果气象数据可用，在顺风/顺流侧（旋回弧扩大侧）额外提前：

```
WOP_wind = 0.1 × WOP_base × (V_wind_cross / 10)     # 横风每 10 m/s 增加 10%
WOP_current = 0.1 × WOP_base × (V_current_cross / 1) # 横流每 1 m/s 增加 10%
```

### 8.3 最终 WOP

```
WOP_total = WOP_base + WOP_rudder + WOP_inertia + WOP_wind + WOP_current
```

此值记录在航点元数据的 `wop_distance` 字段中，供 Guidance 层（L4）在实时跟踪时使用。

### 8.4 WOP 使用方式

Guidance 层在跟踪航线时，当本船与下一个航点的沿航线距离 ≤ WOP_total 时，开始执行转向动作（从当前航向向新航向过渡）。WOP 的精确性直接决定了转弯的质量——过早转弯会切入内侧（over-shoot），过晚转弯会冲出外侧（under-shoot）。

## 9. 步骤五：航点间距校验

### 9.1 最小间距约束

相邻两个转弯航点之间必须有足够的直线段，用于：

1. 完成上一个弯的旋回恢复（回舵、稳定航向）。
2. 完成速度调整（如果下一个弯的 V_safe 与当前段的巡航速度不同）。
3. 为 Guidance 层提供足够的航线跟踪稳定段。

**最小间距公式：**

```
D_min = max(
    A_d(V) × 1.5,                           # 至少 1.5 倍纵距（旋回恢复）
    V_cruise × T_stabilize,                  # 稳定时间 × 巡航速度
    |V_cruise - V_safe_next| × T_decel / 2   # 减速距离（匀减速近似）
)

其中：
  T_stabilize = 180~300 秒（3~5 分钟稳定时间，经验值）
  T_decel = 由 Speed Profiler 计算的减速时间
```

**推荐最小间距经验值：**

| 水域类型 | 最小间距 |
|---------|---------|
| 开阔水域（高速巡航） | ≥ max(500m, 3 分钟 × V_cruise) |
| 沿岸航行 | ≥ max(300m, 2 分钟 × V_cruise) |
| 港内 | ≥ max(100m, 1 分钟 × V_cruise) |

### 9.2 最大间距约束

**推荐最大间距：**

| 水域类型 | 最大间距 |
|---------|---------|
| 开阔水域 | ≤ 50 nm（约 92.6 km） |
| 沿岸航行 | ≤ 15 nm（约 27.8 km） |
| 水深变化剧烈区域 | ≤ 5 nm（约 9.3 km） |
| 港内 | ≤ 1 nm（约 1.85 km） |

### 9.3 间距不满足时的处理

**间距过小：** 合并相邻航点。如果两个航点之间的转角都很小（< 10°），可以保留转角更大的那个航点，删除另一个。如果两个航点的转角都较大，说明航线设计本身有问题，需要触发策略 C 请求重规划。

**间距过大：** 在航段中点位置插入一个直航航点（转角 = 0°），同时检查该位置的水深和安全性。

## 11. 速度剖面生成接口

### 11.1 职责划分

WP Generator 负责确定每个航点处的速度约束（`speed_at_wp` 字段）。Speed Profiler 子模块负责在相邻航点之间生成平滑的加速/巡航/减速曲线。本节定义两者之间的接口。

### 11.2 WP Generator 输出给 Speed Profiler 的约束

对于每个航段（WP_i → WP_{i+1}），WP Generator 提供以下约束：

```
V_max_leg    : 本航段的最大允许速度（由水域类型、限速规定、水深决定）
V_at_wp_i    : 到达 WP_i 时的速度（上一个转弯的出弯速度或加速后的速度）
V_at_wp_i1   : 到达 WP_{i+1} 时的目标速度（下一个转弯的 V_safe）
D_leg        : 航段距离
```

### 11.3 Speed Profiler 生成逻辑

Speed Profiler 根据以上约束生成三段式速度剖面：

```
阶段 1：加速段（如果 V_at_wp_i < V_max_leg）
    从 V_at_wp_i 加速到 V_max_leg
    加速度 a_accel ≤ a_max（最大加速度，由主推进器推力决定）
    加速距离 D_accel = (V_max_leg² - V_at_wp_i²) / (2 × a_accel)

阶段 2：巡航段
    以 V_max_leg 匀速航行
    巡航距离 D_cruise = D_leg - D_accel - D_decel

阶段 3：减速段（如果 V_max_leg > V_at_wp_i1）
    从 V_max_leg 减速到 V_at_wp_i1
    减速度 a_decel ≤ a_decel_max（最大减速度，由主推进器倒车推力或自由减速阻力决定）
    减速距离 D_decel = (V_max_leg² - V_at_wp_i1²) / (2 × a_decel)
```

**关键校验：** D_accel + D_decel ≤ D_leg。如果不满足，说明航段太短无法同时完成加速和减速。此时需要降低 V_max_leg 使得加减速距离之和等于 D_leg。求解方程：

```
V_peak² = (2 × a_accel × a_decel × D_leg + a_decel × V_at_wp_i² + a_accel × V_at_wp_i1²) 
          / (a_accel + a_decel)

V_max_leg_adjusted = sqrt(V_peak²)
```

### 11.4 减速度参数

| 减速方式 | 典型减速度 | 说明 |
|---------|----------|------|
| 自由减速（关闭主机，靠水阻力减速） | 0.01~0.05 m/s² | 非常缓慢，需要大量提前距离 |
| 主动减速（降低推力但不倒车） | 0.05~0.15 m/s² | 常规操作 |
| 倒车减速（倒车推力） | 0.15~0.50 m/s² | 较剧烈，需考虑结构载荷 |
| 紧急倒车（全功率倒车） | 0.30~1.00 m/s² | 仅用于紧急情况 |

**建议：** WP Generator 规划使用的减速度取"主动减速"的值，留有充分余量。紧急倒车作为 Tactical 层的应急手段，不在航线规划阶段使用。

## 12. 浅水效应修正

### 12.1 旋回半径修正

当检测到航线经过浅水区域（h/T < 3.0）时，WP Generator 必须用修正后的旋回参数重新进行转角评估。

**Norrbin 浅水修正公式：**

```
R_shallow / R_deep = (h/T)^(-n)

其中 n 为修正指数：
  n = 1.3~1.7（取决于船型，肥大型船 n 偏大，瘦削型船 n 偏小）
  建议初值 n = 1.5
```

示例：深水中 R_deep = 25 m，水深/吃水比 h/T = 1.5：
R_shallow = 25 × (1.5)^(-1.5) = 25 × 0.544 ≈ 13.6 m ... 这不对，应该是增大。

修正公式应为：

```
R_shallow / R_deep = (h_ref / h)^n    当 h < h_ref

其中 h_ref = 基准深水水深（取旋回试验时的实际水深）
     h     = 当前水域水深
     n     = 1.3~1.7
```

或者使用更直观的修正系数表：

| h/T 比 | 旋回半径修正系数 k_shallow | 应用方式 |
|-------|-------------------------|---------|
| > 3.0 | 1.00 | R_eff = R_deep × k_shallow |
| 2.5~3.0 | 1.10~1.15 | |
| 2.0~2.5 | 1.15~1.30 | |
| 1.5~2.0 | 1.30~1.50 | |
| 1.2~1.5 | 1.50~1.80 | |
| < 1.2 | > 1.80 | 建议避免在此深度转弯 |

### 12.2 舵效修正

浅水中舵效降低，满舵旋回的实际舵力矩下降。经验修正：

```
δ_eff = δ_command × k_rudder_shallow

k_rudder_shallow = 1.0             当 h/T > 3.0
                 = 0.85~0.95       当 h/T 在 1.5~3.0
                 = 0.70~0.85       当 h/T 在 1.2~1.5
```

### 12.3 Squat 效应对 UKC 的影响

浅水中的速度限制不仅影响操纵性，还影响安全水深。Squat 效应导致船体下沉，进一步减小 UKC。

**Barrass 修正公式（受限航道）：**

```
S_max = K × Cb × (V_k / sqrt(h))²

其中：
  S_max = 最大下沉量，米
  K     = 航道系数（开阔水域 K ≈ 2.4，受限航道 K ≈ 2.0~3.0）
  Cb    = 方形系数
  V_k   = 船速，节
  h     = 水深，米
```

WP Generator 在浅水区域应执行以下额外校验：

```
UKC_dynamic = h - T - S_max(V_plan)

IF UKC_dynamic < UKC_min THEN
    降速至 V_squat_safe，使 UKC_dynamic ≥ UKC_min
    V_squat_safe 通过二分搜索求解
```

## 13. 风流影响修正

### 13.1 对旋回轨迹的影响

横风和横流会导致旋回圈变形：

- **顺风/顺流侧**旋回弧扩大（船被吹向/推向外侧）
- **逆风/逆流侧**旋回弧收缩

**修正策略：** 在转角评估中，对旋回空间占用量 Y_max 做方向性修正：

```
# 计算横向风流分量
V_wind_cross = V_wind × sin(wind_dir - heading)      # 横风分量
V_current_cross = V_current × sin(current_dir - heading) # 横流分量

# 总横向外力等效漂移速度
V_drift = V_wind_cross × K_wind + V_current_cross

其中 K_wind 为风压系数，取决于船的受风面积：
  高干舷/集装箱船：K_wind ≈ 0.03~0.05
  低干舷 USV：K_wind ≈ 0.01~0.02

# 风流侧旋回空间修正
Y_max_corrected = Y_max + V_drift × T_turn

其中 T_turn = 旋回时间 ≈ (|Δψ| / 360°) × (2π × R / V_turn)
```

### 13.2 对 WOP 的影响

参见第 8.2 节中 WOP_wind 和 WOP_current 的修正公式。

### 13.3 偏流修正角

风流引起的漂移会导致船的对地轨迹（COG）偏离航向（Heading）。WP Generator 应在航点元数据中记录预估的偏流修正角，供 Guidance 层参考：

```
drift_angle = atan(V_drift / V_cruise)
```

经验值：横流 1 节时偏流角约 3°~8°（取决于船速），横风 20 节时偏流角约 2°~5°。

| 2.0 | 3.9 | 18 | 9 | 24 | 20 | 10.0 | 0.85 |
| 4.0 | 7.8 | 28 | 14 | 36 | 31 | 15.5 | 0.80 |

### 15.3 算例一：90° 转弯，开阔水域

**条件：**
- 巡航速度 V = 8.0 m/s
- 转角 Δψ = 90°
- 可用走廊 W_avail = 40 m
- 水域类型：开阔水域，η = 0.70

**转角评估：**

```
Y_max ≈ T_r(8.0) × (90° / 90°) = 24 × 1.0 = 24 m
W_usable = 40 × 0.70 = 28 m
24 m ≤ 28 m → 可行
```

**WOP 计算：**

```
WOP_base = R(8.0) × tan(90° / 2) = 25.5 × tan(45°) = 25.5 m
WOP_rudder = 8.0 × (2.0 + 7.0) = 72 m     （τ_δ=2s, 满舵建立一半时间=7s）

... 等等，这里需要重新考虑。舵响应时间不应该这么大。

实际上对于USV：
  τ_δ = 1 秒（响应延迟）
  从 0° 到 35° 的建立时间约 7~10 秒
  有效延迟取一半建立时间：t_δ_eff = 5 秒

WOP_rudder = 8.0 × (1.0 + 5.0/2) = 8.0 × 3.5 = 28.0 m
WOP_inertia = 1.5 × 12 = 18 m
WOP_wind = 0（无风假设）

WOP_total = 25.5 + 28.0 + 18.0 = 71.5 m
```

**结论：** 在距离航点 71.5 米处开始打舵。

### 15.4 算例二：120° 转弯，受限水域

**条件：**
- 巡航速度 V = 8.0 m/s
- 转角 Δψ = 120°
- 可用走廊 W_avail = 30 m
- 水域类型：沿岸受限，η = 0.60

**转角评估：**

```
Y_max ≈ T_r(8.0) + (A_d(8.0) - T_r(8.0)) × ((120-90) / 90)
      = 24 + (48 - 24) × 0.333
      = 24 + 8 = 32 m

W_usable = 30 × 0.60 = 18 m
32 m > 18 m → 不可行
```

**策略 A：降速**

```
目标：找到 V_safe 使得 Y_max(120°, V_safe) ≤ 18 m

尝试 V = 4.0 m/s:
  Y_max ≈ T_r(4.0) + (A_d(4.0) - T_r(4.0)) × 0.333
        = 14 + (28 - 14) × 0.333 = 14 + 4.67 = 18.67 m
  18.67 > 18 → 仍不可行

尝试 V = 3.0 m/s（插值）:
  A_d(3.0) ≈ 23, T_r(3.0) ≈ 11.5
  Y_max ≈ 11.5 + (23 - 11.5) × 0.333 = 11.5 + 3.83 = 15.33 m
  15.33 ≤ 18 → 可行

V_safe = 3.0 m/s
```

**Speed Profiler 需要在到达此航点前从 8.0 m/s 减速到 3.0 m/s。**

减速距离（以 0.10 m/s² 主动减速）：

```
D_decel = (8.0² - 3.0²) / (2 × 0.10) = (64 - 9) / 0.2 = 275 m
```

即：需要在距离航点至少 275 m + WOP_total 之前开始减速。

### 15.5 算例三：120° 转弯，极受限水域（需要拆分）

**条件：** 与算例二相同，但 W_avail = 12 m，η = 0.50，W_usable = 6 m。

**策略 A：** 即使降速至 V_min = 2.0 m/s：

```
A_d(2.0) = 18, T_r(2.0) = 9
Y_max ≈ 9 + (18 - 9) × 0.333 = 12 m > 6 m → 仍不可行
```

**策略 B：拆分**

```
目标子转角 Δψ_sub = 30°（极受限水域 ≤ 30°）
n = ceil(120 / 30) = 4 段

V = 2.0 m/s 下每个 30° 子转角的空间占用：
Y_max ≈ T_r(2.0) × (30 / 90) = 9 × 0.333 = 3.0 m
3.0 m ≤ 6.0 m → 每个子转角可行

生成 3 个过渡航点，将 120° 转弯拆分为 4 个 30° 转弯。
```

## 17. 实现建议与测试策略

### 17.1 开发优先级

建议按以下顺序开发子模块，每完成一个子模块即可在仿真中验证。

**Phase 1（第一优先级）：** Turn Evaluator + WOP Calculator。这两个模块解决的是最急迫的问题——高速冲弯。完成后在仿真中验证：给定不同转角和速度，WP Generator 能否正确判断可行性并计算 WOP。

**Phase 2：** Speed Constraint Calculator + Turn Splitter。完成后验证：对于不可行的转弯，系统能否自动降速或拆分。

**Phase 3：** ENC Query + Special Zone Handler。完成后验证：航线能否自动避开禁区和浅水区，进出港能否自动加密航点。

**Phase 4：** Spacing Validator + 全流程集成。完成后验证：全航线的端到端生成。

## 1. 概述

本文档定义 MASS_ADAS 在 45m Crew Boat 驾驶台上的全部硬件配置——从传感器到显示器、从操纵杆到紧急停船按钮、从计算节点到 UPS 电源。

硬件总体分为四个物理区域：主操控台（前方，面向窗户）、侧翼操控台（右舷，靠泊用）、设备机柜（后方角落）、顶部传感器桁架（monkey island）和船壳传感器。

## 4. 操控设备

### 4.1 伺服电动推杆（Motorized Command Levers）

| 参数 | 规格 |
|------|------|
| 数量 | 2 套（左推进器 + 右推进器），每套含推力杆和方向杆 |
| 行程 | 推力：200mm（0~100%），方向：±60° |
| 伺服电机 | 24V DC 伺服，20W，力矩 ≥ 2 Nm |

| 机械离合器 | 脱开力 ~50N（伺服锁死时紧急释放） |
| 通信 | CAN Bus（读取 AI 推力值）+ 模拟量（到 Arbiter） |
| 控制器 | 独立 MCU（STM32 级别），不依赖 ROS2 |
| 防护 | IP54 |
| 安装 | 操控台中央，船长座椅两侧 |
| 参考文档 | Hardware Override Arbiter 文档第 14 节 |

### 4.2 舵轮

| 参数 | 规格 |
|------|------|
| 类型 | 传统船用舵轮（NFU/FU 双模式） |
| 直径 | 300~350mm |
| 反馈 | 舵角指示器（机械式 + 电子式） |

### 4.3 IJS 独立摇杆面板（Independent Joystick System）

| 参数 | 规格 |
|------|------|
| 摇杆 | 三轴工业摇杆（前后=Surge, 左右=Sway, 旋转=Yaw） |
| 处理器 | 独立 MCU（STM32/Arduino Mega 级别） |
| 通信 | CAN Bus 直连推进器控制器（不经过 ROS2/以太网） |
| 推力分配 | 内置简化线性分配算法 |
| 激活方式 | 主控心跳丢失 3 秒自动激活，或手动按钮激活 |
| 电源 | 独立 24V DC（独立于主 UPS） |
| 状态指示 | LED：蓝闪=IJS 激活 / 绿=主控正常 / 红=全部故障 |
| 安装 | 操控台左侧独立面板，物理隔离于主操控台布线 |
| 参考文档 | Mode Switcher 文档 IJS_EMERGENCY 章节 |

### 7.5 近场相对定位系统（Position Reference System）

45m Crew Boat 频繁靠离码头和海上平台——港内 GNSS 多径干扰严重（高架桥、起重机、建筑物反射），位置可能跳变数米。必须标配独立于 GNSS 的近场定位系统。

| 参数 | 规格 |
|------|------|
| 类型 | RADius 微波雷达测距（推荐）或 SpotTrack 激光测距 |
| 数量 | 1 套 |
| 原理 | RADius: C-band 微波测距目标反射器；SpotTrack: 激光测距棱镜 |
| 精度 | RADius: ±0.1m / SpotTrack: ±0.01m |
| 作用距离 | RADius: 1000m / SpotTrack: 500m |
| 全天候 | RADius: 全天候（微波不受雾影响）/ SpotTrack: 受大雾影响 |
| 输出 | 距离 + 方位 → 经 Navigation Filter EKF 融合 |
| 接口 | 以太网或 RS-422 |
| 目标端 | 在常用泊位和平台上安装被动反射器（一次性安装成本低） |
| 安装 | 传感器桁架前方 |
| 选型参考 | Kongsberg RADius 2000 / Kongsberg SpotTrack |
| 说明 | 建议 RADius（全天候优势）。靠泊时自动激活，与 GNSS 并行融合。详见 Navigation Filter 文档近场定位源章节 |

| 存储 | 环形缓冲 30 分钟（高频 AI 语义日志）+ 事件触发永久存档 |

## 10. 电源与配电

### 10.1 UPS 不间断电源

| 参数 | 规格 |
|------|------|
| 类型 | 在线式 UPS |
| 容量 | 3 kVA（支撑全部计算和显示设备） |
| 后备时间 | ≥ 15 分钟（全船断电后维持系统运行） |
| 输入 | 船舶 AC 380V/220V |
| 输出 | AC 220V + DC 24V |
| 安装 | 设备机柜底部 |

### 10.2 电源分配

| 供电路径 | 供电设备 | 来源 |
|---------|---------|------|
| 路径 A | 计算节点 A + 显示器 1/2 + 传感器组 1 | 主配电板 → UPS A |
| 路径 B | 计算节点 B + 显示器 3/4 + 传感器组 2 | 应急配电板 → UPS B |
| 路径 C | IJS 独立摇杆 + Override 硬件 | 应急配电板 → 独立 24V DC |
| 路径 D | Emergency Stop | 推进器电源回路直接接入 |

**关键设计：** 路径 A 和路径 B 完全独立——任一路径断电不影响另一路径。IJS 和 Override 使用独立于 UPS 的 24V DC 供电——即使 UPS 全部故障，IJS 和 Override 仍然工作。

## 11. 布线与安装要求

### 11.1 布线原则

| 原则 | 说明 |
|------|------|
| 强弱电分离 | 电源线缆与信号线缆分槽/分管，间距 ≥ 50mm |
| IT/OT 物理隔离 | IT 网络线缆（卫星/4G）与 OT 网络线缆在不同线槽中 |
| 传感器独立走线 | GPS/罗经的 RF 同轴电缆远离发动机和大电流线路 |
| IJS 独立走线 | IJS 的 CAN Bus 和模拟量线缆独立于主控计算机的以太网线缆 |
| 冗余路径分离 | 路径 A 和路径 B 的线缆在物理上分开的线槽中——防止单一火灾或机械损伤同时切断两条路径 |

### 11.2 接地与屏蔽

| 要求 | 说明 |
|------|------|
| 单点接地 | 传感器信号线缆的屏蔽层在交换机端单点接地 |
| 隔离接地 | 计算节点的信号接地与船体接地隔离（防止地环路噪声） |
| 雷击保护 | GPS/VHF/卫星天线的同轴电缆安装防雷器 |

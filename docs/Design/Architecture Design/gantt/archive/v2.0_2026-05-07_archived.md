## Phase 1 · 工程基础设施 + SIL 测试框架

**时间**：2026-05-01 → 2026-06-15（6.5周）
**估计人周**：~7人周（2人并行）
**阶段目标**：在写任何业务模块代码之前，先建立"测试基础设施"和"质量门"。没有可运行的 CI 和 SIL，任何模块都无法验证质量。SIL 框架是整个项目的技术基础，**最高优先级**。

> ★ **注意**：5/1–5/5 劳动节假期，实际开工 5/6。

### SIL 框架说明（独立优先级模块）

**SIL（Software-in-the-Loop）框架**是本项目的测试基础设施核心，Phase 1 优先完成，Phase 2–4 全程依赖。包含以下子模块：

| 子模块 | 说明 |
|---|---|
| FCB 4-DOF MMG 物理仿真器 | 船体运动（x/y/ψ/r），Yasukawa 2015 标准参数，初始 YAML 参数（HAZID 校准前）|
| AIS 历史数据管道 | NMEA 0183 解析 → TrackedTargetArray；支持 NOAA/ChAIS 数据集；可配置回放速率 1×/5×/10× |
| YAML 声明式场景管理 | 定义初始状态（ownship + N个目标），自动运行仿真，输出结果 JSON；支持批量 + CI 集成 |
| COLREGs 覆盖率自动报告 | Rule 5/6/7/8/9/13-17/19 × 通过/失败矩阵；自动生成 HTML 报告（格式满足 CCS 提交要求）|
| SIL 调试 HMI | 实时显示 SAT-1（当前状态）/ SAT-2（决策依据）/ SAT-3（预测）；手动场景注入；ODD 状态可视化 |
| RL 对抗性场景生成（测试工具）| 训练 RL 智能体寻找令当前 TDL 系统出错的边界场景；生成场景加入场景库驱动修复 |

> **RL 定位说明**：RL 仅作为**测试工具**使用（对抗场景生成），**不作为**生产决策算法——CCS 认证要求 COLREGs 决策白盒可审计，RL 黑箱无法通过认证。

D1.1
开发环境验证报告
📅 第 1 周末👤 基础设施负责人
交付内容
ROS2 + 仿真平台（推荐 Gazebo/Unity）安装验证文档；45m FCB MMG 水动力基础模型跑通记录；CI/CD 流水线配置说明；Hello World 级联调测试截图/视频。
验收标准（DoD）
FCB 单船仿真连续运行 10 分钟无崩溃
控制回路频率达到 50 Hz（M5 目标频率）
CI/CD 可自动执行单元测试并生成报告
文档通过技术负责人审核
战略价值：确保后续所有开发活动可正常进行的基础条件
验收：技术负责人
D1.2
接口契约文档 v1.0
📅 第 3 周末👤 系统架构师
交付内容
8 个模块全部输入/输出消息的 IDL 定义；消息频率要求表；置信度（confidence）字段规范；时间戳规范；版本控制说明。此文档冻结后，模块间接口不允许无评审变更。
验收标准（DoD）
8 个模块的全部消息类型有完整 IDL 定义（≥12 条消息）
每条消息包含：数据字段 + 频率 + 置信度 + 时间戳
技术评审会议通过（各模块负责人签字确认）
纳入版本管理，后续变更走变更控制流程
战略价值：防止并行模块开发时接口不兼容导致集成阶段大量返工
验收：架构师+各负责人
D1.3
Capability Manifest 规格书 v1.0（FCB 45m）
📅 第 3 周末👤 系统架构师
交付内容
45m FCB 的 YAML 格式 Capability Manifest 文件及说明文档；包含：主尺度、操纵极限（ROT_max、旋回半径、制动距离）、ODD 各子域参数、水动力插件接口规范。此文件须经船厂确认后送 CCS 项目组。
验收标准（DoD）
Manifest 文件可被 M1 正确加载并通过 Schema 验证
ODD-A/B/C/D 四个子域的关键参数全部填写
FCB 船型参数经船厂设计部门书面确认
水动力插件接口完成第一版 C++ 接口定义
战略价值：多船型扩展的核心文件，也是 CCS 入级 AIP 申请的必要输入
验收：架构师+CCS
D1.4
M1 ODD/Envelope Manager 实现及测试报告
📅 第 4 周末👤 M1 负责人
交付内容
M1 模块完整实现代码（含三轴状态机、TMR/TDL 实时计算、ODD 边界检测、D2/D3/D4 模式切换、MRC 触发逻辑）；单元测试套件；测试覆盖率报告；10 个标准 ODD 场景测试记录。
验收标准（DoD）
单元测试覆盖率 ≥ 80%（分支覆盖）
10 个标准 ODD 场景（各子域切换 + MRC 触发）全部通过
ODD_StateMsg 发布频率稳定在 1 Hz ± 10%
代码审查（Code Review）通过，无阻塞性问题
与 M2 联调：从 M2 威胁摘要正确触发 ODD 边界警告
战略价值：整个系统的调度枢纽，M3–M8 开发的先决依赖模块
验收：技术负责人
D1.5
M2 World Model 实现及测试报告
📅 第 4 周末👤 M2 负责人
交付内容
M2 模块完整实现代码（含三视图数据结构、Mock 传感器接口、ENC 查询接口、目标 COLREGs 几何预分类、置信度管理）；单元测试套件；与 M1 的联调测试记录。
验收标准（DoD）
单元测试覆盖率 ≥ 80%
仿真中 World_StateMsg 稳定以 4 Hz 发布
Mock AIS 接入：10 目标场景下 CPA/TCPA 计算误差 ≤ 1%
ENC 查询接口：给定坐标能正确返回水深和禁区信息
M1 + M2 联调：威胁 TCPA 变化正确触发 ODD 状态变更
战略价值：所有决策模块（M3–M6）的唯一事实来源，后续所有模块强依赖
验收：技术负责人
D1.6
ODD-Spec v1.0（CCS AIP 核心申请文件）
📅 第 4 周末👤 系统架构师
交付内容
Operational Envelope 三轴（E/T/H）完整定义文档；四个 ODD 子域（开阔/狭水道/港内/能见度不良）的参数表；TMR/TDL 量化基础说明；ODD 主状态机规格；格式满足 CCS 提交要求。
验收标准（DoD）
三轴 E/T/H 定义完整，每轴各维度有量化范围
四个 ODD 子域参数表完整（CPA/TCPA/速度/TMR/主导规则）
文档格式通过内部技术评审
已提交 CCS 项目组进行 AIP 前预沟通（邮件留存）
战略价值：CCS i-Ship 入级 AIP（原则认可）的核心申请材料，启动认证流程的起点
验收：架构师+CCS征询
D1.7
HARA 风险分析初版（含 SIL 候选分配）
📅 第 4 周末👤 安全工程师
交付内容
基于 8 模块架构的顶层危险识别（≥30 个 Hazard）；危险等级初步评估（严重度 × 频率）；SIL 候选分配表（MRC 触发链 SIL 2 等）；关键假设清单。后续 HAZID Workshop 将以此为输入细化。
验收标准（DoD）
识别 ≥ 30 个顶层危险，覆盖所有 8 个模块
每个危险有：描述 / 严重度 / 频率 / 初步风险等级
SIL 候选分配覆盖 MRC 链路和 COLREGs 推理模块
通过内部安全评审（安全工程师+技术负责人）
战略价值：SIL 分配的前提，也是 CCS 认证阶段安全论证（Safety Case）的基础文件
验收：安全工程师
D1.8
M3–M8 框架代码（全链路心跳验证）
📅 第 4 周末👤 技术负责人
交付内容
M3（Mission Manager）、M4（Behavior Arbiter）、M5（Tactical Planner）、M6（COLREGs Reasoner）、M7（Safety Supervisor）、M8（HMI Bridge）六个模块的框架代码；接口完整（有正确的订阅和发布），业务逻辑为占位实现（返回合法的默认值）。
验收标准（DoD）
全部 8 个模块可在仿真环境中同时启动，无启动报错
8 模块全链路心跳测试：M1 发出 odd_state → 各模块正确接收
M5 能发出格式合法的 Trajectory_CmdMsg（即使是占位值）
代码结构符合第 2 阶段正式实现的扩展要求
战略价值：第二阶段并行开发的前提条件，确保各模块负责人可立即开始第 2 阶段工作
验收：技术负责人

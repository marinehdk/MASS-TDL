## 1. 页面概述

### 1.1 定位
预检页面是仿真启动前的最后检查站，完成系统自检、ENC 加载验证、场景签名确认后进入 Captain HMI。

### 1.2 布局原则
- **居中式设计**：5 项自检居中，专注检查
- **无地图**：预检阶段不需要海图显示
- **倒计时过渡**：自检完成后 3-2-1 倒计时进入仿真

## 3. 自检项目详情

### 3.1 ① ENC 加载验证

| 项目 | 说明 |
|------|------|
| 检查内容 | S-57/S-63 ENC tiles 是否可加载 |
| 验证方法 | 尝试解析 ENC tile metadata |
| 通过条件 | 所有必需 tile 可访问 |
| 失败处理 | 显示缺失 tile 列表 + 建议 |
| 显示信息 | Tile 数量 / 覆盖范围 / 加载时间 |

**状态样式**：
- ⏳ 检查中：灰色 + 旋转图标
- ✓ 通过：绿色 + 勾号
- ✗ 失败：红色 + 叉号 + 错误详情

### 3.2 ② ASDR 启动确认

| 项目 | 说明 |
|------|------|
| 检查内容 | ASDR（船载数据记录器）服务是否就绪 |
| 验证方法 | POST `/sil/self_check/probe` |
| 通过条件 | ASDR 响应 < 100ms |
| 失败处理 | 显示错误码 + 重试按钮 |
| 显示信息 | 启动延迟 / 缓冲区状态 |

### 3.3 ③ UTC 同步

| 项目 | 说明 |
|------|------|
| 检查内容 | 系统时钟与 NTP/PTP 同步状态 |
| 验证方法 | 读取 `/clock` 偏移量 |
| 通过条件 | drift < 10ms（DNV-RP-0513 要求）|
| 失败处理 | 显示偏差值 + 警告（可跳过）|
| 显示信息 | UTC 偏移 / 同步源 / 同步状态 |

### 3.4 ④ M1-M8 模块健康检查

| 项目 | 说明 |
|------|------|
| 检查内容 | L3 TDL Kernel 8 模块运行状态 |
| 验证方法 | 订阅 `/l3/m{1-8}/health` topic 5s |
| 通过条件 | 所有模块 5s 内有响应 |
| 失败处理 | 高亮失败模块 + 建议 |
| 显示信息 | M1-M8 状态徽章（GREEN/AMBER/RED）|

**模块状态徽章**：

| 模块 | 状态徽章 | 说明 |
|------|---------|------|
| M1 ODD | 🟢 GREEN | ODD 管理器健康 |
| M2 WM | 🟢 GREEN | 世界模型健康 |
| M3 MIS | 🟢 GREEN | 任务管理器健康 |
| M4 BEH | 🟢 GREEN | 行为仲裁器健康 |
| M5 PLN | 🟢 GREEN | 战术规划器健康 |
| M6 COL | 🟢 GREEN | COLREGs 推理器健康 |
| M7 SAF | 🟢 GREEN | 安全监督器健康 |
| M8 HMI | 🟢 GREEN | HMI 桥健康 |

### 3.5 ⑤ 场景签名确认

| 项目 | 说明 |
|------|------|
| 检查内容 | 场景 YAML SHA256 签名 |
| 验证方法 | 计算 YAML SHA256 与存储值比对 |
| 通过条件 | 签名匹配 + 文件完整 |
| 失败处理 | 警告：场景被篡改 |
| 显示信息 | SHA256 前8位 / 签名时间 / 签名者 |

## 4. 倒计时阶段

### 4.1 触发条件
所有 5 项自检通过后，自动进入倒计时。

### 4.2 倒计时设计

```
┌────────────────────────────────────────┐
│                                        │
│                 3                      │  ← 大号数字居中
│                                        │
│         ████████████░░░░░░░░          │  ← 进度条
│                                        │
│         系统将在 3 秒后启动仿真          │
│                                        │
└────────────────────────────────────────┘
```

**倒计时选项**：
- 自动继续（默认）
- 暂停倒计时（用户手动点击"开始"）
- 取消（返回场景构建器）

### 4.3 取消机制
点击"取消"按钮中断倒计时，返回预检完成状态。

## 5. 操作按钮

| 按钮 | 位置 | 行为 | 状态 |
|------|------|------|------|
| 返回场景构建器 | 左下 | Router push `/builder` | 始终可用 |
| 跳过预检 | 左下 | 跳过未通过项，直接开始 | 需要确认弹窗 |
| 开始运行 | 右下 | Router push `/bridge/:runId` | 仅全部通过后可用 |

**跳过预检确认弹窗**：
```
┌────────────────────────────────────────┐
│ 确认跳过预检？                         │
│                                        │
│ 以下项目未通过验证：                   │
│ • UTC 同步偏移 15ms（建议修复）        │
│                                        │
│ 跳过预检可能导致仿真异常。             │
│ 确认继续？                             │
│                                        │
│        [取消]        [确认跳过]        │
└────────────────────────────────────────┘
```

## 6. 进度指示

### 6.1 单项进度
每项自检独立显示进度：
- 初始：❓ 等待
- 运行中：⏳ 检查中（旋转图标）
- 完成：✓ / ✗

### 6.2 总体进度
底部 Overall Status 实时更新：
- "5/5 项检查完成" 或 "3/5 项通过，2 项失败"

## 7. 状态管理（Zustand）

```typescript
// usePreflightStore
interface PreflightStore {
  runId: string;
  
  // 自检状态
  checks: {
    encLoading: CheckStatus;
    asdrStartup: CheckStatus;
    utcSync: CheckStatus;
    moduleHealth: ModuleHealthStatus[];
    scenarioSignature: CheckStatus;
  };
  
  // 倒计时状态
  countdown: number;
  isCountingDown: boolean;
  
  // 操作
  startChecks: () => Promise<void>;
  cancelCountdown: () => void;
  proceedToBridge: () => void;
}

type CheckStatus = 'pending' | 'running' | 'passed' | 'failed';

interface ModuleHealthStatus {
  moduleId: 'M1' | 'M2' | 'M3' | 'M4' | 'M5' | 'M6' | 'M7' | 'M8';
  status: 'GREEN' | 'AMBER' | 'RED' | 'UNKNOWN';
  latencyMs: number;
}
```

## 8. API 接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 启动自检 | POST | `/api/v1/lifecycle/self_check/start` | 开始 5 项自检 |
| 自检状态 | GET | `/api/v1/lifecycle/self_check/status` | 返回各项检查结果 |
| ENC 验证 | POST | `/api/v1/enc/validate` | 验证 ENC tile 可用性 |
| ASDR probe | POST | `/api/v1/sil/self_check/probe` | ASDR 启动确认 |
| UTC 状态 | GET | `/api/v1/sil/clock/status` | UTC 同步状态 |
| 模块健康 | GET | `/api/v1/l3/modules/health` | M1-M8 健康状态 |
| 场景签名 | GET | `/api/v1/scenarios/{id}/signature` | YAML SHA256 签名 |
| 配置仿真 | POST | `/api/v1/lifecycle/configure` | 配置并准备仿真 |
| 激活仿真 | POST | `/api/v1/lifecycle/activate` | 激活仿真（倒计时后）|

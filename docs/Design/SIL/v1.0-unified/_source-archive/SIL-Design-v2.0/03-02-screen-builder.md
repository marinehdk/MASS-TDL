## 1. 页面概述

### 1.1 定位
场景构建器是 SIL 框架的全局调度控制台，它决定了后续系统是在做 Debug（单场景验证）、压测（蒙特卡洛）还是训练模型（强化学习）。为支撑大规模验证，其左侧管理树划分为**三大运行域 (Operation Domains)**：
1. **单一场景 (Single Scenarios)**：1v1确定性验证快照。
2. **蒙特卡洛实验 (Monte Carlo)**：基于场景模版衍生出的批量测试任务（包含参数分布与执行次数）。
3. **深度学习 (RL Playground)**：交互式动态生成器环境（Phase 4 特性）。

### 1.2 布局原则
- **Master-Detail 结构**：左侧多级文件树管理域，右侧为基底场景的参数化蓝图工作台。
- **参数动态适配**：右侧表单随左侧选中的运行域，在“固定绝对值”与“概率分布区间”之间动态变形。

### 1.3 顶部导航

```
└─────────────────────────────────────────────────────────────────────────────┘
```

## 3. 左侧区域：运行域多级文件树管理

### 3.1 顶层：运行域切换器
位于左上角的 Segmented Control，用于切换不同的业务模态：
- **单一场景 (Single Scenarios)**：展示独立的、确定性验证的场景模版列表。
- **蒙特卡洛实验 (Monte Carlo)**：展示基于场景模版衍生出的批量测试任务集，例如 `IMAZU-14 鲁棒性验证 (100 runs)`。
- **RL 深度学习 (Playground)**：展示持续交互的训练环境生成器配置。

### 3.2 树状资源管理器 (Explorer)
抛弃了原有的卡片平铺设计，采用**三层文件树架构**以应对海量衍生场景：

| 层级 | 说明 | 示例 |
|------|------|------|
| **1. 套件 (Suite)** | 相当于文件夹，按测试目的或海域聚类。 | `📁 IMAZU 22 Benchmark`、`📁 Trondheim Fjord 测试` |
| **2. 基底场景 (Blueprint)** | 核心几何逻辑模板，右侧工作台对应它的配置。 | `📄 Imazu-01 Head-on` |
| **3. 实验变体 (Permutation)**| 仅在蒙特卡洛/RL域显示，折叠为参数空间摘要。 | `↳ 100 Runs (Seeds 1~100)` 或 `↳ 环境风速: Uniform[5, 10]` |

**交互操作**：
- 支持拖拽、分组归类。
- 悬停浮现快捷操作：复制（用于创建变体）、导出 YAML、删除。

## 4. 右侧区域

### 4.1 右上：ENC 地图预览

**功能**：
- 显示选中 ENC 区域的矢量海图
- 叠加 Imazu-22 遭遇几何可视化（虚线圆圈）
- 本船位置（红色三角）
- 目标船位置（蓝色三角）
- 航线（蓝色虚线）

**交互**：
- 缩放/平移
- 点击地图设定本船/目标船初始位置
- Imazu 场景类型选择（自动切换几何预览）

**图层切换**：
- ENC 矢量（默认）
- 卫星底图
- OpenSeaMap
- 地形起伏

### 4.2 右下：6 大参数化配置表单

表单支持根据左侧选中的“运行域”动态变形（固定输入 vs 分布区间）。

#### Tab 1：基本配置
| 字段 | 类型 | 说明 |
|------|------|------|
| 场景/实验名称 | 文本输入 | 全局唯一标识 |
| 遭遇类型 | 下拉选择 | Head-on / Crossing / Overtaking / 自定义 |
| 航行区域 | 下拉选择 | 挂载的底层 ENC 区域 |
| 仿真控制 | 组块 | 总时长、RK4步长；在蒙特卡洛模式下，增加 N 次运行数量设置 |

#### Tab 2：本船配置
| 字段 | 类型 | 动态参数化支持 |
|------|------|----------------|
| 船型模型 | 下拉选择 | 不可变 (FCB / ASV) |
| 初始坐标 | 经纬度输入 | 不可变 (蒙特卡洛下必须为固定锚点) |
| 初始航速与航向 | 数字输入 | 蒙特卡洛下支持选择区间偏差 `[±X kn]`, `[±Y°]` |

#### Tab 3：目标船配置
| 字段 | 类型 | 动态参数化支持 |
|------|------|----------------|
| 运动控制模式 | 下拉选择 | ais_replay / ncdm / intelligent |
| 相对方位/距离生成 | 极坐标输入 | 在 RL 模式下可配置为“生成器”边界范围多边形 |
| 初始航速与航向 | 数字输入 | 蒙特卡洛下支持概率分布（如高斯/均匀分布） |

#### Tab 4：环境配置
| 字段 | 类型 | 动态参数化支持 |
|------|------|----------------|
| 风/流模型 | 下拉选择 | gauss_markov / constant / impulse |
| 强度与方向 | 数字输入 | 蒙特卡洛下极易变参数，支持输入均值与方差范围 |
| 海况与能见度 | 下拉选择 | 蒙特卡洛下可设置为批量离散覆盖测试 |
| Random Seed | 整数 | 在蒙特卡洛下变为 Range (如 `1-100`) |

#### Tab 5：传感器配置
| 字段 | 类型 | 动态参数化支持 |
|------|------|----------------|
| 跟踪器模式 | 下拉选择 | God (完美感知) / KF (卡尔曼滤波后验数据) |
| AIS / Radar 故障引入 | 滑块 / 开关 | 蒙特卡洛下可配置掉线率/噪声分布的均方差 |

#### Tab 6：AIS 数据导入
提供对历史真实海量 AIS 数据集的抽取能力，通过时空边界框 (Bounding Box) 筛选并提取出具有挑战性的真实碰撞遭遇切片。

| 字段 | 类型 | 说明 |
|------|------|------|
| AIS 数据源 | 文件上传 | Kystverket / NOAA CSV |
| 时间范围 | 日期时间选择 | 开始–结束 |
| 地理边界框 | 坐标输入 | 左上/右下经纬度 |
| DCPA 阈值 | 数字输入 | 500m |
| TCPA 阈值 | 数字输入 | 10 min |

**[抽取遭遇场景]** 按钮

## 5. 操作按钮

```
┌─────────────────────────────────────────┐
│ [保存]  [验证 YAML]  [运行 →]          │
└─────────────────────────────────────────┘
```

| 按钮 | 行为 | 反馈 |
|------|------|------|
| 保存 | POST /scenarios + YAML 写入 | Toast: "场景已保存" |
| 验证 YAML | maritime-schema + cerberus 校验 | Toast: "验证通过" / 错误列表 |
| 运行 → | 跳转预检页面 + 创建 run_id | Router push `/preflight/:runId` |

## 6. ENC 预览地图功能

### 6.1 地图功能

| 功能 | 说明 |
|------|------|
| 矢量海图渲染 | S-57 MVT 格式 |
| 缩放范围 | 0-18 级 |
| 本船 sprite | heading-up，居中，红色三角 |
| 目标船 sprite | 蓝色三角，带 COG 矢量 |
| 航线显示 | 蓝色虚线 waypoint 连接 |
| 遭遇几何 | Imazu-22 虚线圆圈（可选）|

### 6.2 图层切换

| 图层 | 数据源 | 用途 |
|------|--------|------|
| ENC 矢量 | S-57/S-63 MVT | 航海图 |
| 卫星底图 | Mapbox/ESRI | 直观参考 |
| OpenSeaMap | OSM Marine | 快速预览 |
| 地形起伏 | DEM | 浅水深标注 |

### 6.3 遭遇几何可视化

当选择 Imazu-22 场景类型时，自动叠加：
- DCPA 圆（虚线）
- 遭遇扇区（彩色填充）
- 相对距离/方位标注

```typescript
// useScenarioStore
interface ScenarioStore {
  // 场景列表
  scenarios: Scenario[];
  selectedScenarioId: string | null;
  
  // 操作
  loadScenarios: () => Promise<void>;
  selectScenario: (id: string) => void;
  createScenario: () => void;
  saveScenario: (scenario: Scenario) => Promise<void>;
  deleteScenario: (id: string) => Promise<void>;
  
  // 地图状态
  mapCenter: [lat, lon];
  mapZoom: number;
```

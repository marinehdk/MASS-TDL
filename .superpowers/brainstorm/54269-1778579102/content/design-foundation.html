Design Section 1: Color System & Design Tokens

Foundation layer — OpenBridge 5.0 aligned + S-52 Night palette adapted for our dark-themed SIL HMI

Why This Matters

Current code has hardcoded colors scattered across BridgeHMI.tsx (e.g., #0b1320, #2dd4bf, #f87171, #fbbf24). No systematic color token system = inconsistent screens, hard to maintain. OpenBridge 5.0 + IHO S-52 PresLib 4.0 give us industry-standard tokens.

Size Scale

text-xs: 10px — HUD values, tooltips
text-sm: 11-12px — body, panel text
text-base: 13-14px — headings, KPIs
text-lg: 16px — sim time, verdict badge

Why this works

  • OpenBridge 5.0 colors = defendable to DNV/CCS auditors
  • S-52 layer hierarchy = correct ECDIS rendering order
  • Monospace for data + sans for chrome = clear visual separation
  • Single CSS custom properties file → all screens consistent

Risks

  • Dark-only theme = no Day/Dusk/Night toggle for Demo-1
  • OpenBridge is designed for light backgrounds — our night adaptation is a derivative, not certified
  • CSS custom properties need to work with MapLibre's GL JSON style spec (not CSS)
